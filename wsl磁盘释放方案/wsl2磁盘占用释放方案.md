# WSL2 磁盘占用释放方案：删了几十G文件，磁盘为什么还没回来？

**WSL2 很香。**

作为一个 Linux 环境，WSL2 不仅解决了 Windows 用户的 Linux 开发体验问题，更是广大 **“爱折腾”的技术爱好者、HomeLab 玩家和数字极客**的快乐老家。

再搭配 Docker：

**WSL2 + Docker，简直是折腾党的数字玩具库。**

不管你是写代码配置环境（Java、Node、Python），还是折腾各种本地服务——跑本地 AI 大模型（Ollama、ComfyUI）、挂载媒体服务、搭建自动化工具链……各种镜像随便拉、容器随便起，随用随销，爽得飞起。

但是，用久了之后，你会发现一个非常离谱的问题：

**文件删了，磁盘空间没回来。**

---

## WSL2：你删你的，我占我的

平时折腾的过程中，我们经常会做这些事情：

* 拉取各种 Docker 镜像
* 尝鲜各种开源工具与容器服务
* 删除不用的容器与废弃镜像
* 清理模型缓存、编译日志与临时文件
* 删除 `node_modules` 与各类依赖

Linux 里面看：

```bash
df -h
```

好像空间已经释放了。

但是回到 Windows，一看你的 WSL2 虚拟磁盘：

`ext4.vhdx` 大小纹丝不动。

甚至你会发现：**删掉几十 GB 数据之后，WSL2 占用的空间反而越来越大。**

这时候你可能会怀疑：*我是不是哪里删错了？*

其实没有。这是 WSL2 虚拟磁盘的机制导致的。

WSL2 本质上使用的是一个动态扩容的虚拟硬盘文件（`ext4.vhdx`）。它可以随着 Linux 内部数据增长自动变大，**但是它不会因为 Linux 删除文件而自动缩小。**

所以结果就是：**Linux 觉得空间释放了，Windows 觉得这块盘还在占着。**

---

## SSD 越来越贵，这几十 G 真的不能忍

以前磁盘空间大，几十 GB 可能无所谓。

现在 SSD 价格贵的离谱，特别是笔记本用户。

一个 WSL2，50GB、100GB、200GB 慢慢膨胀。关键是你里面可能已经没有这么多数据了。

看着这些空间白白被占用，多少有点难受。

那么有没有办法让它真正释放回来？

有。缩容的方式有两种：

* **Sparse VHD**
    * 适用于新版本的wsl，新功能，社区反馈不稳定,这里不做说明

* **手动diskpart(稳定可靠，亲测可用)**
    1. 在Linux 里面擦除已删文件的痕迹（防止碎片）
        ```bash
        sudo fstrim -av
        ```
    2. 彻底关闭 WSL
        ```powershell
        wsl --shutdown
        ```
    3. 找到你的wsl的ext4.vhdx文件
    4. 用 Windows 自带的 diskpart 压缩（管理员身份运行 PowerShell）
        ```powershell
        diskpart
        select vdisk file="你的ext4.vhdx路径"
        compact vdisk
        exit
        ```

---

## 效果怎么样？

以我自己的使用场景来说，WSL2 + Docker 长时间折腾后：

* 镜像和开源项目来来回回拉
* 容器与服务不停创建删除
* 各种缓存与日志不断增加

最后 `ext4.vhdx` 经常膨胀到几十甚至上百 GB。

执行一次 `fstrim` + `diskpart compact`，空间会明显下降。

那种感觉非常奇妙：**磁盘释放了感觉钱又回来了真好**。

---

## 总结

WSL2 很香，WSL2 + Docker 更香。

但是 WSL2 的虚拟磁盘机制也带来了一个问题：**删除文件 ≠ 释放 Windows 磁盘空间。**

如果你的 WSL2 使用时间比较久，尤其是 Docker 与开源服务重度折腾用户，建议定期检查一下 `ext4.vhdx` 大小。

不用第三方软件，不用重新安装环境。几条命令，就能把那些“消失”的磁盘空间找回来。

毕竟：**SSD 的每一个 GB，都是钱。**
